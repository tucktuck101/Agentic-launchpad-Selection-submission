# Task 1 Code Review Plan

## Objective

Create reviewer-facing code review notes for the Express/React support app without fixing app code.

## Paths

- Task folder: `challenge_two/task_1_code_review/`
- App folder: `challenge_two/task_1_code_review/code-review-app/`
- Required artefact: `challenge_two/task_1_code_review/code-review-notes.md`

## Before Writing

- Read root `AGENTS.md`, `challenge_two/PLAN.md`, this file, and `Agents.md`.
- Read the Code Review brief in `Challenge_info.md`.
- Inspect existing task files and any existing review notes before creating or replacing artefacts.
- Inspect both server and client code in `code-review-app/`.

## Definition Of Done

- `code-review-notes.md` explains what the app does.
- It lists at least three bugs, risks, or design concerns.
- It names the first change to make and why.
- It names what to leave alone for now.
- It lists extra information that would help the review.
- It includes evidence paths, inspected files, and any commands run.
- No app code is changed for this review task.

## Boundaries

- Review only; do not fix app code.
- Do not complete other Part 2 tasks.
- Do not edit `../../agentic-launchpad-selection-submission.md` or `../../Original_Files/`.
